# LidLock 1.0.1

Maintenance release for LidLock on macOS.

## Download

The attached `LidLock.dmg` is signed with Apple Developer ID and notarized by Apple.

Homebrew:

```sh
brew install --cask imwalkinhere/lidlock/lidlock
```

SHA-256:

```text
d4e2e1b97ffcf1315f8915a9519ea328dffefde65bd02f3ccd9726aa0ed44c47  LidLock.dmg
```

## What Changed

- Fixed a restore-on-battery policy path that could ask for admin approval a
  second time immediately after enabling Survive Lid Close while on battery.
- Kept explicit Survive Lid Close enables from being auto-undone during the
  same battery session.
- Improved menu/window state consistency while privileged operations are in
  progress.
- Fixed Launch at Login cleanup when macOS leaves the login item in a pending
  approval state.

## Requirements

- macOS 13 Ventura or newer.
- A MacBook for lid-close workflows. Other Macs can still use keep-awake and
  display controls.

## Privacy

LidLock has no accounts, analytics, telemetry, advertising, or automatic network
access during normal operation.

Privacy policy: https://delpageinc.com/privacy/lidlock
